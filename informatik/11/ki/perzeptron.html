Teil 1 – Vom Gehirn zum Modell

Nervenzellen im Gehirn leiten Impulse nur weiter, wenn ein Schwellwert überschritten wird. Viele schwache Signale können zusammen stark genug sein – das Gehirn summiert also Eingangsreize und entscheidet dann.

Genau dieses Prinzip steckt hinter dem künstlichen Neuron (Perzeptron): mehrere Eingangsgrößen werden gewichtet addiert, das Ergebnis (Nettoeingabe $z$) mit einem Schwellwert verglichen und daraus eine binäre Ausgabe erzeugt.

💡 Modell des künstlichen Neurons

Ein Neuron besitzt drei Eingaben:

  • Konstante Verschiebung: Eingabe $= 1$, Gewicht $w_0$ (Bias)
  • Merkmal $x_1$: z. B. Länge Kelchblatt, Gewicht $w_1$
  • Merkmal $x_2$: z. B. Länge Kronblatt, Gewicht $w_2$

Die Nettoeingabe berechnet sich als:

$$z = w_0 \cdot 1 + w_1 \cdot x_1 + w_2 \cdot x_2$$

Die Ausgabe (Heaviside-Funktion) ist:

$\text{Ausgabe} = \begin{cases} 1 & \text{falls } z \geq 0 \\ 0 & \text{falls } z < 0 \end{cases}$

🖼️ Struktur des Perzeptrons

Der Bias-Eingang (konstant 1) verschiebt die Entscheidungsgrenze – ähnlich dem y-Achsenabschnitt einer Geraden.

💡 Was ist die Entscheidungsgrenze?

Die Entscheidungsgrenze ist die Menge aller Punkte $(x_1, x_2)$, für die gilt $z = 0$, also: $$w_0 + w_1 \cdot x_1 + w_2 \cdot x_2 = 0$$ Aufgelöst nach $x_2$ ergibt sich eine lineare Funktion (Gerade) im Merkmalsraum: $$x_2 = -\frac{w_1}{w_2} \cdot x_1 - \frac{w_0}{w_2}$$

Durch Verändern der Gewichte dreht und verschiebt man diese Gerade.

💬 Tipp für Aufgabe 1c – Hinweis einblenden

Die Daten sind linear trennbar! Beobachte: Die setosa-Punkte haben deutlich kürzere Kronblätter. Versuche $w_2$ positiv und groß zu halten, und $w_0$ leicht negativ einzustellen, um die Grenze zwischen den Punktgruppen zu platzieren. Zum Beispiel: $w_0 \approx -10$, $w_1 \approx 0$, $w_2 \approx 1$.

Teil 4 – Die Delta-Lernregel

Bislang haben wir die Gewichte manuell eingestellt. Das Ziel ist nun, das Perzeptron automatisch zu trainieren. Dazu brauchen wir eine Regel, die die Gewichte schrittweise verbessert.

💡 Idee der Delta-Lernregel

Nach jedem Datensatz wird der Fehler $(y_i - \text{Ausgabe})$ betrachtet:

  • Vorhersage korrekt: Fehler $= 0$ → keine Gewichtsänderung
  • Vorhersage zu klein (0 statt 1): Fehler $= +1$ → Gewichte erhöhen
  • Vorhersage zu groß (1 statt 0): Fehler $= -1$ → Gewichte verringern

Formal: Lernrate $\eta$ (eta) steuert die Schrittgröße:

$$\Delta w_0 = \eta \cdot (y_i - \text{Ausgabe})$$ $$\Delta w_1 = \eta \cdot (y_i - \text{Ausgabe}) \cdot x_1^{(i)}$$ $$\Delta w_2 = \eta \cdot (y_i - \text{Ausgabe}) \cdot x_2^{(i)}$$

📋 Pseudocode der Delta-Lernregel

Wiederhole anzahlEpochen mal:
  Für i von 0 bis < Anzahl Trainingsdatensätze:
    // Nettoeingabe berechnen
    z = w0 + w1 * x1[i] + w2 * x2[i]
    // Schwellwertfunktion
    falls z >= 0: ausgabe = 1
    sonst:         ausgabe = 0
    // Gewichte anpassen (nur wenn Fehler ≠ 0)
    w0 = w0 + η * (y[i] - ausgabe)
    w1 = w1 + η * (y[i] - ausgabe) * x1[i]
    w2 = w2 + η * (y[i] - ausgabe) * x2[i]

Ein vollständiger Durchlauf aller Datensätze heißt Epoche. Nach endlich vielen Epochen konvergiert das Verfahren (wenn die Daten linear trennbar sind).

⚠️ Wahl der Lernrate η

η zu groß: Die Gewichte schwingen über das Ziel hinaus (Oszillation) und konvergieren möglicherweise nicht.
η zu klein: Das Lernen dauert sehr viele Epochen – aber es ist stabiler.
Typisch: $\eta \in [0{,}01,\ 1{,}0]$. Standard in diesem Beispiel: $\eta = 0{,}1$.

💡 Grenzen des Perzeptrons

Das Perzeptron kann nur linear trennbare Probleme lösen – es erzeugt immer eine gerade Entscheidungsgrenze. Daten, die z. B. kreisförmig verteilt sind oder das XOR-Problem darstellen, kann ein einzelnes Perzeptron nicht korrekt klassifizieren.

Abhilfe schaffen mehrschichtige neuronale Netze (Multilayer Perceptrons), die beliebig komplexe Entscheidungsgrenzen erzeugen können – das Thema der nächsten Einheit.

🔑 Musterlösung – Delta-Lernregel in Java

📂 Musterlösung einblenden (erst nach eigenem Versuch!) 
for (int epoche = 0; epoche < anzahlEpochen; epoche++) {
    double w0_alt = w_0, w1_alt = w_1, w2_alt = w_2;
    for (int i = 0; i < x_1.length; i++) {
        // Nettoeingabe
        double z = w_0 + w_1 * x_1[i] + w_2 * x_2[i];
        // Ausgabe (Schwellwert)
        int ausgabe = (z >= 0) ? 1 : 0;
        // Fehler
        int fehler = y[i] - ausgabe;
        // Gewichtsanpassung
        w_0 = w_0 + eta * fehler;
        w_1 = w_1 + eta * fehler * x_1[i];
        w_2 = w_2 + eta * fehler * x_2[i];
    }
    println("Epoche " + (epoche+1) + ": w0=" + w_0 + ", w1=" + w_1 + ", w2=" + w_2);
    // Konvergenzerkennung
    if (w_0 == w0_alt && w_1 == w1_alt && w_2 == w2_alt) {
        println("Konvergenz nach Epoche " + (epoche+1) + " – keine weiteren Änderungen.");
        break;
    }
}

Zusammenfassung

✅ Das haben wir gelernt

1
Das Perzeptron modelliert ein biologisches Neuron: gewichtete Summe der Eingaben, dann Schwellwertentscheidung.
2
Die Nettoeingabe $z = w_0 + w_1 x_1 + w_2 x_2$ erzeugt eine lineare Entscheidungsgrenze.
3
Die Delta-Lernregel passt die Gewichte automatisch an den Fehler an – datensatzweise, in mehreren Epochen.
4
Die Lernrate η steuert die Schrittgröße und beeinflusst Geschwindigkeit und Stabilität.
5
Ein Perzeptron löst nur linear trennbare Probleme – für komplexere braucht man mehrere Schichten.
