Modul 1

Grundlagen der Netzwerkkommunikation

📡 Was ist ein Netzwerk?

Wenn Computer Daten miteinander austauschen, bilden sie ein Netzwerk. Computer sind heute in den unterschiedlichsten Gegenständen wie Smartphones, Tablets, Smart-TVs oder IoT-Geräten präsent.

📊 Netzwerkgrößen

🏠

PAN

Personal Area Network

Persönliches Netzwerk (z.B. Bluetooth)

Reichweite: ~10m

🏢

LAN

Local Area Network

Lokales Netzwerk (z.B. Heimnetzwerk, Schule)

Reichweite: ~100m

🏙️

WAN

Wide Area Network

Weitverkehrsnetz (z.B. Stadtwerke)

Reichweite: mehrere km

🌍

Internet

Worldwide Network

Weltweites Netzwerk

Reichweite: global

👥 Client-Server Prinzip

Ein Client stellt eine Anfrage nach einem bestimmten Dienst, der von einem Server angeboten wird.

💻 Client

Stellt Anfrage

z.B. Webbrowser

→

🖥️ Server

Liefert Antwort

z.B. Webserver

💡 Beispiel:

Wenn du eine Website aufrufst, ist dein Browser der Client und der Computer, der die Website bereitstellt, ist der Server.

Modul 2

Netzwerk-Topologien

🔗 Verschiedene Topologien im Überblick

Zur Vernetzung der einzelnen Geräte zu einem Netzwerk gibt es verschiedene Möglichkeiten:

🚌

Bus-Topologie

Alle Geräte sind an einem gemeinsamen Übertragungsmedium (Bus) angeschlossen.

Vorteile:
  • Einfacher Aufbau
  • Kostengünstig
  • Wenig Kabel
Nachteile:
  • Kabelbruch = Totalausfall
  • Kollisionen möglich
  • Schwer erweiterbar
Anwendung: Kleine Netze, ISPs
⭐

Stern-Topologie

Alle Geräte sind mit einem zentralen Knoten (Switch/Hub) verbunden.

Vorteile:
  • Ausfall eines Geräts = kein Problem
  • Einfache Erweiterung
  • Keine Kollisionen
Nachteile:
  • Zentraler Knoten = Single Point of Failure
  • Viel Kabel nötig
  • Teurer als Bus
Anwendung: Heimnetzwerke, Büros
🔄

Ring-Topologie

Geräte sind in einem geschlossenen Ring verbunden. Daten fließen in eine Richtung.

Vorteile:
  • Keine Kollisionen
  • Gleichmäßige Datenübertragung
  • Vorhersagbare Performance
Nachteile:
  • Ausfall eines Geräts = Problem
  • Schwer zu erweitern
  • Komplexe Fehlersuche
Anwendung: Früher in Universitäten
🕸️

Vermaschte Topologie

Jedes Gerät ist mit mehreren anderen Geräten verbunden.

Vorteile:
  • Sehr ausfallsicher
  • Alternative Routen
  • Hohe Redundanz
Nachteile:
  • Sehr teuer
  • Viel Kabel
  • Komplexe Verwaltung
Anwendung: Mesh-WiFi, kritische Infrastruktur

Modul 3

Adressierung von Geräten

🏷️ IP-Adressen: Die Postanschrift im Internet

Jeder Rechner in einem Netzwerk erhält eine eindeutige IP-Adresse (Internet Protocol).

IPv4

Aktueller Standard

Format: 4 Blöcke von 0-255 (je 1 Byte)

Beispiel: 192.168.0.1

Anzahl Adressen:

2⁸ × 2⁸ × 2⁸ × 2⁸ = 2³²

= 4.294.967.296

≈ 4,3 Milliarden

IPv6

Zukunft

Format: 8 Blöcke von 0-65535 (je 2 Byte)

Beispiel: 2001:0db8:85a3::8a2e:0370:7334

Anzahl Adressen:

(2¹⁶)⁸ = 2¹²⁸

≈ 3,4 × 10³⁸

💡 Mehr als Sterne im Universum!

⚠️ Das Adressierungsproblem bei IPv4

Da IPv4-Adressen nicht ausreichen, teilen sich mehrere Geräte eine öffentliche IP. Im Heimnetzwerk vergibt der Router private IPs über DHCP.

🔌 Ports und MAC-Adressen

🔌

Port-Nummern

Da Geräte mehrere Anwendungen parallel nutzen, wird ein Port verwendet (0-65535).

Notation: IP:Port

88.231.4.72:8080

  • HTTP: 80
  • HTTPS: 443
  • FTP: 21
  • SSH: 22
🔧

MAC-Adresse

Jedes Netzwerkgerät besitzt eine eindeutige Hardware-Adresse.

Format: 6 Byte

2a:03:7a:e8:32:0f

⚠️ Wichtig:

Nicht sicher zur Identifizierung (MAC-Spoofing möglich)

DNS und URL-Struktur

🌍 DNS - Das Telefonbuch des Internets

Meist geben wir nicht die IP ein, sondern eine URL. Der DNS-Server wandelt diese in IPs um.

🔗 Aufbau einer URL

Protokoll
https://
Legt fest, wie Daten übertragen werden (HTTP, HTTPS, FTP, ...)
Subdomain
www.
Optional: Unterbereich der Domain (z.B. blog., shop., mail.)
Domain
example.com
Hauptadresse der Website
Pfad
/products/shoes
Spezifische Seite oder Ressource auf dem Server
Parameter
?size=42&color=blue
Zusätzliche Informationen für den Server

🔄 Ablauf einer Website-Anfrage

1
Eingabe im Browser

Du gibst "www.example.com" ein

2
DNS-Anfrage

Browser fragt DNS-Server: "Welche IP hat example.com?"

3
DNS-Antwort

DNS-Server antwortet: "93.184.216.34"

4
HTTP-Anfrage

Browser sendet Anfrage an IP 93.184.216.34

5
Server-Antwort

Webserver sendet HTML, CSS, JavaScript zurück

6
Rendering

Browser zeigt die Website an

Das TCP/IP-Schichtenmodell

📚 Kommunikation in Schichten

Die Kommunikation ist in Schichten aufgeteilt. Jede hat eine spezielle Aufgabe.

🏗️ Die 4 Schichten des TCP/IP-Modells

4. Anwendungsschicht (Application Layer)

HTTP, FTP, SMTP, DNS, SSH
Anwendungsspezifische Protokolle für Browser, E-Mail, Dateiübertragung

3. Transportschicht (Transport Layer)

TCP, UDP
Segmentierung der Daten, Ports, Fehlerkorrektur

2. Internetschicht (Internet Layer)

IP (IPv4, IPv6)
Adressierung und Routing zwischen Netzwerken

1. Netzzugangsschicht (Network Access Layer)

Ethernet, WLAN
Physische Übertragung der Daten (Kabel, Funk)

⚔️ TCP vs. UDP

TCP

Zuverlässig
  • ✅ Verbindungsorientiert
  • Garantierte Zustellung
  • Reihenfolge wird eingehalten
  • Fehlerkorrektur
  • Langsamer (Overhead)
Verwendung:

HTTP, FTP, E-Mail

UDP

Schnell
  • ⚡ Verbindungslos
  • Keine Zustellgarantie
  • Reihenfolge egal
  • Keine Fehlerkorrektur
  • Schneller (weniger Overhead)
Verwendung:

Streaming, Gaming, DNS

🎮 Übung: Paketgröße berechnen

Ein Foto hat 2,5 MB. Die maximale Paketgröße beträgt 65.535 Byte.

Frage: Wie viele Pakete werden benötigt?

💡 Lösung:

1 MB = 1.048.576 Byte

2,5 MB = 2.621.440 Byte

Anzahl Pakete = 2.621.440 ÷ 65.535 = 40 Pakete (aufgerundet)

Modul 6

HTTP, APIs und Client-Implementierung

🌐 HTTP-Methoden: GET vs. POST

GET-Request

Daten abrufen
  • Fordert Daten an
  • Parameter in URL sichtbar
  • Kann gecacht werden
  • Im Browser-Verlauf
  • Idempotent (wiederholbar)
Beispiel:

GET /search?q=netzwerke

POST-Request

Daten senden
  • Sendet Daten zum Server
  • Parameter im Body (unsichtbar)
  • Wird nicht gecacht
  • Nicht im Verlauf
  • Nicht idempotent
Verwendung:

Formulare, Login, Uploads
