Schritt 1

Der Ablauf

Kein Technik-Stress

Du musst dich nicht um Server, CORS-Header oder CSS-Dateien kümmern. Das System ist zentralisiert.

  • Erstelle eine einfache HTML-Datei (lokal auf deinem PC).
  • Verwende die im Abschnitt IDE Einbindung stehenden Links für CSS und JS.
  • Teste die Seite grob in deinem Browser.
  • Sende die fertige .html Datei an contribute@edu-mrh.de.
Schritt 2

Die Technik

Basis-Setup

Damit deine Seite im einheitlichen Design erscheint und die IDE funktioniert, kopiere diesen Block in den <head> deiner Datei:

<!-- Community Styles -->
<link rel="stylesheet" href="https://edu-mrh.de/embed/style.css">

<!-- Online IDE Core -->
<link rel="stylesheet" href="https://edu-mrh.de/embed/include/online-ide-embedded.css">
<script type="module" src="https://edu-mrh.de/embed/include/online-ide-embedded.js"></script>

Der IDE-Block

Füge diesen Code dort im <body> ein, wo das Editor-Fenster erscheinen soll:

<div class="java-online" style="height: 400px;" 
     data-java-online='{"id": "UniqueName1", "speed": 500000, "withBottomPanel": true}'>
    <script type="text/plain" title="Programm.java">
public class Start {
    public static void main(String[] args) {
        System.out.println("Hallo Welt");
        // Dein Übungscode hier...
    }
}
    </script>
</div>

📥 style.css herunterladen 📥 script.js herunterladen 
Du bist ein Experte für die Erstellung von Unterrichtsmaterialien als Webseiten. Erstelle eine vollständige, produktionsreife HTML-Seite für eine Unterrichtseinheit zum Thema [THEMA HIER EINFÜGEN] für die Jahrgangsstufe [JAHRGANGSSTUFE] im Fachbereich [FACHBEREICH].

TECHNISCHE ANFORDERUNGEN (STRIKT EINHALTEN):

1. DOKUMENTSTRUKTUR:
- Erstelle eine valide HTML5-Seite mit vollständigem DOCTYPE, head und body
- Setze die Sprache auf Deutsch: <html lang="de">
- Verwende UTF-8 Encoding: <meta charset="UTF-8">
- Füge Viewport-Meta-Tag hinzu: <meta name="viewport" content="width=device-width, initial-scale=1.0">
- Setze einen aussagekräftigen <title> im Format: "[Thema] - [Fachbereich] - edu-mrh.de"
- Füge eine Meta-Description hinzu (max. 160 Zeichen)
- Füge am Anfang des body folgende Metadaten als Kommentar ein:
<!--
Fachbereich: [FACHBEREICH]
Jahrgangsstufe: [STUFE]
Thema: [THEMA]
Erstellt: [AKTUELLES DATUM]
Version: 1.0
-->

2. CSS-FRAMEWORK (PFLICHT - im <head> einbinden):
<link rel="stylesheet" href="https://edu-mrh.de/embed/style.css">

3. JAVASCRIPT-FRAMEWORK (PFLICHT - im <head> einbinden):
<script defer src="https://edu-mrh.de/embed/script.js"></script>

4. FAVICON (PFLICHT):
<link rel="icon" type="image/png" href="https://edu-mrh.de/embed/icons/favicon.png">

5. NAVIGATION (PFLICHT):
Erstelle eine Navigation mit folgender Struktur:
<nav class="navbar">
    <div class="container">
        <div class="nav-brand">
            <span class="icon">[PASSENDES ICON AUS LISTE WÄHLEN]</span>
            <a href="https://edu-mrh.de/">edu-mrh.de</a>
        </div>
        <div class="nav-links">
            <a href="https://edu-mrh.de/[FACHBEREICH]/[FACHBEREICH].html#jahrgang-[JAHRGANGSSTUFE]" class="btn btn-outline">Übersicht</a>
            <a href="https://edu-mrh.de/contribute.html#impressum" class="btn btn-outline">Impressum</a>
        </div>
    </div>
</nav>

ICON-AUSWAHL (wähle das passendste):
- 🖥️ für Informatik
- 🔢 für Mathematik
- ⚛️ für Physik
- 🧪 für Chemie
- 🌱 für Biologie
- 📚 für Sprachen
- 🌍 für Geographie/Geschichte
- 🎨 für Kunst/Kreatives

ERSETZE in der Navigation:
- [FACHBEREICH] durch den konkreten Fachbereich in Kleinbuchstaben (z.B. "informatik", "mathematik")
- [JAHRGANGSSTUFE] durch die Zahl (z.B. "7", "10", "11")

6. KATEX FÜR MATHEMATIK (NUR bei mathematischen Inhalten):
Falls mathematische Formeln benötigt werden, binde im <head> ein:
<link rel="stylesheet" href="https://edu-mrh.de/embed/katex/katex.min.css">
<script defer src="https://edu-mrh.de/embed/katex/katex.min.js"></script>
<script defer src="https://edu-mrh.de/embed/katex/contrib/auto-render.min.js" onload="renderMath()"></script>

Nutze für Formeln:
- Inline: $Formel$ (z.B. $x^2 + y^2 = z^2$)
- Display: $$Formel$$ (z.B. $$\\int_0^\\infty e^{-x^2} dx$$)

7. ONLINE-IDE FÜR JAVA (NUR bei Programmieraufgaben):
Falls Java-Code-Editor benötigt wird, binde im <head> ein:
<link rel="stylesheet" href="https://edu-mrh.de/embed/include/online-ide-embedded.css">
<script type="module" src="https://edu-mrh.de/embed/include/online-ide-embedded.js"></script>

EDITOR-EINBINDUNG im body:
<div class="java-online" style="width: 100%; height: 500px;" data-java-online='{ "id": "eindeutige-id-hier", "speed": 500000, "withBottomPanel": true, "withFileList": true, "withConsole": true }'>
    <script type="text/plain" title="Hauptklasse.java">
public class Hauptklasse {
    public static void main(String[] args) {
        // Code hier
    }
}
    </script>
    <script type="text/plain" title="ZweiteKlasse.java">
public class ZweiteKlasse {
    // Weitere Klasse hier
}
    </script>
</div>

WICHTIG für IDs: Verwende sprechende, eindeutige IDs im Format: [thema]-[aufgabe]-[nummer] (z.B. "schleifen-uebung-01")

LAYOUT UND STRUKTUR:

HAUPTCONTAINER (umschließt gesamten Inhalt):
<div class="container">
    <!-- Gesamter Inhalt hier -->
</div>

KAPITEL-HEADER:
<div class="chapter-header">
    <span class="chapter-number">[Nummer]</span>
    <h1>[Kapiteltitel]</h1>
</div>

CONTENT-BOXEN (verwende passende Typen):

INFO-BOX (für Erklärungen, Definitionen):
<div class="info-box">
    <h3>💡 Wichtig</h3>
    <p>Inhalt hier...</p>
</div>

TASK-BOX (für Aufgaben):
<div class="task-box">
    <h3>✏️ Aufgabe [Nummer]</h3>
    <p>Aufgabenstellung...</p>
</div>

WARNING-BOX (für Warnungen, häufige Fehler):
<div class="warning-box">
    <h3>⚠️ Achtung</h3>
    <p>Warnung...</p>
</div>

INHALTLICHE ANFORDERUNGEN:

STRUKTUR DER UNTERRICHTSEINHEIT:
1. Einleitung (Motivation, Lernziele)
2. Theorieteil (Erklärungen mit Beispielen)
3. Praktische Übungen (mindestens 3 Aufgaben mit steigendem Schwierigkeitsgrad)
4. Zusammenfassung (Kernpunkte wiederholen)
5. Weiterführende Links (optional)

DIDAKTISCHE PRINZIPIEN:
- Klare, verständliche Sprache (altersgerecht)
- Schrittweise Steigerung der Komplexität
- Konkrete Beispiele vor abstrakten Konzepten
- Visuelle Unterstützung durch Boxen und Strukturierung
- Interaktive Elemente (bei Programmierung: lauffähiger Code)
- Selbstkontrolle ermöglichen (Lösungshinweise)

CODE-QUALITÄT (bei Programmieraufgaben):
- Vollständiger, lauffähiger Code
- Sinnvolle Variablennamen (deutsch oder englisch, konsistent)
- Kommentare für komplexe Stellen
- Best Practices der Sprache beachten
- Fehlerbehandlung wo sinnvoll

QUALITÄTSSICHERUNG - PRÜFE VOR ABSCHLUSS:
- Alle Ressourcen korrekt eingebunden
- Navigation vollständig ausgefüllt (keine Platzhalter wie [FACHBEREICH])
- Alle IDs eindeutig
- Code syntaktisch korrekt
- Rechtschreibung und Grammatik
- Responsive Design (funktioniert auf mobilen Geräten)
- Accessibility (alt-Texte, semantisches HTML)

AUSGABEFORMAT:

KRITISCH: Gib den VOLLSTÄNDIGEN HTML-Code aus. Die Datei muss:
- Sofort einsetzbar sein (keine Platzhalter wie "..." oder "Code hier")
- Alle Sektionen vollständig enthalten
- Valides HTML5 sein
- Mindestens 200 Zeilen Code umfassen (für eine vollständige Unterrichtseinheit)

WICHTIG: Beende deine Antwort ERST, wenn ALLE Codezeilen vollständig ausgegeben wurden. KEINE Abkürzungen, KEINE Auslassungen, KEIN "Rest analog".

DATEINAME: Nenne am Ende den empfohlenen Dateinamen im Format: [thema-mit-bindestrichen].html (Beispiel: java-schleifen-einfuehrung.html) - nur Kleinbuchstaben, Bindestriche statt Leerzeichen, keine Umlaute, maximal 50 Zeichen.

Erstelle jetzt die vollständige Webseite.
